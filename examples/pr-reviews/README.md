# Example PR Reviews for Jules

This directory contains examples of how human maintainers can provide feedback on pull requests opened by Jules.

## The Goal of Review

The goal of reviewing a Jules PR is to ensure:
- The implementation matches the issue scope.
- The code quality meets project standards.
- Proper validation (tests, logs) has been performed.
- The engineering history remains clean and understandable.

## Examples

- [Approve](./approve.md): When the PR is ready to merge.
- [Request Changes](./request-changes.md): When the implementation needs minor adjustments.
- [Validation Needed](./validation-needed.md): When the PR lacks sufficient proof of testing or verification.

## Best Practices

- **Be Specific:** Point to specific lines of code.
- **Maintain Identity:** Treat Jules as an AI agent, not a human colleague.
- **Focus on Scope:** Ensure Jules hasn't drifted from the original issue.
- **Ask for Validation:** If you aren't sure it works, ask Jules to provide logs or screenshots.
