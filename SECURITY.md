# Security Policy

## Supported Versions

This repository provides documentation, workflow templates, and case studies for leading an AI coding agent. Security support focuses on the current contents of the `main` branch.

| Version | Supported          |
| ------- | ------------------ |
| `main`  | :white_check_mark: |
| Others  | :x:                |

## Reporting a Vulnerability

**Please do not report security-sensitive issues (such as workflow exploits, prompt injection risks that lead to unauthorized repository access, or leaked credentials) via public GitHub issues.**

Instead, please report any security issues privately by sending an email to the maintainers or using GitHub's private vulnerability reporting feature if enabled.

**What to include in your report:**
- A description of the vulnerability.
- Steps to reproduce the issue.
- Any potential impact on the repository or its users.

**What not to include in public issues:**
- Do not disclose sensitive details, proof-of-concept exploits, or detailed attack vectors in public issue trackers.

## Response Expectations

We take security seriously and will review your report as quickly as possible. However, please note that this is an open-source starter kit maintained on a volunteer basis. While we strive to acknowledge reports promptly, we cannot guarantee immediate responses or unrealistic resolution timelines.

## Review of Security-Related Changes

Any changes to security policies or security-related workflow automation must be reviewed and merged by human maintainers. Jules or any automated agent will not merge security-related pull requests autonomously.
