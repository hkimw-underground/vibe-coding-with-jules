## Longer announcement post

I have been building an AI coding workflow starter kit for Jules.

Most AI coding demos show only the final code. This project focuses on the process: how to scope issues, hand work to Jules, review pull requests, keep CI in the loop, and preserve human maintainer ownership.

The repository includes:

- copy-first starter files
- GitHub Issue and PR templates
- `AGENTS.md` rules for Jules-assisted work
- workflow tracks from Review-Driven to experimental Full Vibe Coding
- a Jules web UI guide
- translation and first-contribution guides
- case study documentation

It is built for students, teachers, first-time maintainers, and developers who want AI-assisted coding work to leave a readable GitHub history.

## Wording to avoid

Avoid wording that overclaims what the project does or misrepresents Jules.

Do not say:

- "Jules does everything for you."
- "No human review is needed."
- "This replaces maintainers."
- "AI built the whole project."
- "Fully autonomous coding is safe by default."
- "Just write a prompt and merge the result."

Prefer:

- "Human maintainers own review and merge decisions."
- "Jules assists through GitHub-native workflows."
- "CI and review gates stay in the loop."
- "The project shows the process, not only the final code."
- "Experimental workflows belong in sandbox repositories or protected branches."

## Positioning checklist

Before sharing the project, check that the copy keeps these points clear:

- Jules is an AI coding agent, not a human contributor.
- Human maintainers own architecture, validation, review, and merge decisions.
- Review-Driven is the recommended default workflow.
- Full Vibe Coding is experimental.
- The repository provides reusable templates and examples, not a promise of autonomous correctness.
