# Translation Guide

We explicitly welcome translation contributors to help make the AI Coding Workflow Starter Kit accessible to a global audience.

## Translation Principles

### 1. English is the Source
English (`.md`) is the source language for all public documentation. Updates to the workflow or core guides should happen in English first.

### 2. Korean as a Companion
Korean (`.ko.md`) is currently maintained as a companion language for the core documentation. We aim to keep the English and Korean versions structurally aligned.

### 3. Accuracy Over Style
The primary goal of translation is to keep the technical meaning and workflow instructions accurate. While natural phrasing is encouraged, do not rewrite the workflow or change the intended meaning of the engineering practices.

### 4. Jules is an AI Agent
In all translations, **Jules must remain described as an AI coding agent**, not a human contributor. Avoid using gendered pronouns or personification that suggests Jules has human agency.

## Technical Requirements

- **Links:** Keep all relative links (e.g., `[Link](./path/to/file.md)`) pointing to the same relative paths.
- **Filenames:** Do not translate filenames mentioned in the text (e.g., `README.md`, `AGENTS.md`) unless they refer to a translated version of that file.
- **Code Blocks:** Keep code blocks, terminal commands, and configuration snippets unchanged unless they contain comments that require translation for understanding.
- **Structure:** Maintain the same heading levels and overall document structure as the English source.

## Suggested First Tasks

If you are looking to contribute a translation, here are some recommended starting points. **To propose a new translation or improve an existing one, please use the [Translation Issue Template](../../.github/ISSUE_TEMPLATE/translation.yml).**

1. **Translate the Quickstart Guide:** Help new users get started by translating `docs/quickstart.md`.
2. **Translate Operations Guides:** Choose a guide from `docs/operations/` and provide a translated version.
3. **Audit Existing Translations:** Review `README.ko.md` and suggest improvements for clarity or accuracy.
4. **Propose a New Language:** If you would like to translate the starter kit into a new language, please open a translation issue first to discuss the structure (e.g., using `.es.md`, `.fr.md`, etc.).

Thank you for helping us reach more developers!
