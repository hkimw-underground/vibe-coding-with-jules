# Translation Guide

Thank you for your interest in translating the AI Coding Workflow Starter Kit for Jules! We want this playbook to be useful for a global audience.

The project currently supports and maintains:
- **English** (`.md`) - The source language for public docs.
- **Korean** (`.ko.md`) - A companion language maintained alongside English.

Future translations into other languages are welcome.

## Guidelines for Translators

### 1. Accurate Technical Meaning
When translating, ensure you keep the technical meaning accurate. Do not rewrite or modify the core GitHub-native workflow or project architecture concepts. The goal is to make the existing process accessible in other languages, not to create a new process.

### 2. Jules is an AI Agent
Jules must remain described as an AI coding agent, not as a human contributor. Maintain the distinction between human maintainers (who make decisions and review) and Jules (who assists).

### 3. Preserving Structure
Translation PRs should keep the following unchanged unless strictly necessary for grammatical correctness in the target language:
- Links (URLs)
- Filenames
- Code blocks and configuration snippets
- Issue and PR templates structure

### 4. Getting Started
Suggested first translation tasks:
1. Translate the `README.md` to introduce the project.
2. Translate the `docs/quickstart.md` to help users get started quickly.
3. Translate the `docs/tracks/review-driven.md` as it is the recommended default workflow.
