# Showcase

## What this is
Most AI coding demos show only the final code. This project shows the process.

## Who it is for
This starter kit is for students, teachers, first-time maintainers, and small teams who want AI-assisted coding work to leave a readable engineering history.

## Copy these files first
To start using this workflow in your own repository, copy these files and directories:

- **[AGENTS.md](../AGENTS.md)**: Rules and context that Jules follows while working in your repo.
- **[.github/ISSUE_TEMPLATE/](../.github/ISSUE_TEMPLATE/)**: Standardized templates for tasks and workflow experiments.
- **[.github/PULL_REQUEST_TEMPLATE.md](../.github/PULL_REQUEST_TEMPLATE.md)**: Ensures every PR links to an issue and includes validation notes.
- **[.github/workflows/docs-and-templates.yml](../.github/workflows/docs-and-templates.yml)**: Automated checks to maintain repository hygiene and structure.
- **[prompts/](../prompts/)**: Reusable prompts for handoffs and status reporting.
- **[examples/](../examples/)**: Real-world examples of how to write issues and review PRs.

## Default workflow
The default **Review-Driven** workflow ensures quality through human oversight:

`Issue → Jules task → Pull request → CI → Human review → Merge`

## Choose a track
Select the track that matches your goals:

- **Review-Driven**: The recommended default for real projects and classrooms.
- **GitHub-Native Collaboration**: Focused on learning GitHub features like labels and branch protection.
- **Half Vibe Coding**: An advanced, low-touch track for experienced maintainers.
- **Full Vibe Coding**: An experimental sandbox for testing new AI workflows.
