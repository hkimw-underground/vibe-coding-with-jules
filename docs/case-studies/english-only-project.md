# Case Study B: English-Only Open Source Project

## Status: Planned

This case study is currently planned and will be added in a future update.

## Overview

The goal of this case study is to demonstrate a clean, English-only project workflow for a global audience. It will focus on:

- Small, well-scoped GitHub Issues.
- Focused Pull Requests implemented by Jules.
- CI-first development practices.
- Readable and professional review comments in an open-source context.

## Expected Outcomes

This case study will show how the Jules workflow can be applied to standard open-source projects outside of specific educational or capstone contexts, maintaining high standards of documentation and engineering history.
